# Nepal Housing Market Dashboard

## Setup — Creating a Virtual Environment

A virtual environment keeps all dependencies isolated to this project.

**Step 1** — Create a virtual environment:

```
python -m venv venv
```

**Step 2** — Activate the virtual environment:

On **Windows**:

```
venv\Scripts\activate
```

On **Mac / Linux**:

```
source venv/bin/activate
```

You should see `(venv)` appear at the start of your terminal line — this means the environment is active.

**Step 3** — Install all required libraries:

```
pip install pandas numpy plotly jinja2
```

---

## How to Run the Dashboard

Make sure the virtual environment is activated before running these commands.

**Step 1** — Generate all charts and KPI data:

```
python data_preparation.py
```

This creates `chart_data.json` in the project folder.

**Step 2** — Build the final dashboard:

```
python generate_dashboard.py
```

This reads `chart_data.json` and `dashboard_template.html`, combines them, and saves `final_dashboard.html`.

**Step 3** — Open the dashboard in any browser:

```
dashboard.html
```

Simply double-click the file or drag it into a browser window.

---

## Files

| File                       | Description                                                        |
| -------------------------- | ------------------------------------------------------------------ |
| `Nepali_house_cleaned.csv` | Dataset (2,128 rows)                                               |
| `data_preparation.py`      | Step 1 — generates all 12 charts and KPIs, saves `chart_data.json` |
| `generate_dashboard.py`    | Step 2 — builds the final dashboard, saves `final_dashboard.html`  |
| `dashboard_template.html`  | HTML template used by Step 2 (do not open directly)                |
| `chart_data.json`          | Auto-generated by Step 1 (do not edit manually)                    |
| `dashboard.html`           | **Final output — open this file in your browser**                  |

---

## Notes

- All files must be kept in the same folder.
- All commands must be run from inside the project folder with the virtual environment active.
- `dashboard.html` is fully self-contained — no internet connection or Python installation is needed to view it once generated.
- Do not open `dashboard_template.html` directly in a browser — it will not render correctly.
